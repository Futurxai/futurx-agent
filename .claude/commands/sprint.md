# /sprint command
When the user types /sprint:
1. Read ROADMAP.md
2. Identify current phase (first phase with unchecked items)
3. List all unchecked items in that phase
4. Suggest the top 3 tasks for this week based on:
   - Dependencies (what must be done first)
   - Team availability
   - Impact on other phases
5. Output as sprint table: Task | Owner | Est. Hours | Blocker | Package
