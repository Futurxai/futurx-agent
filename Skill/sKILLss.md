## Product Thinking Framework

Always think through these 3 lenses:

```
DESIRABILITY  → Do users actually want this?
VIABILITY     → Is it good for the business?
FEASIBILITY   → Can engineering build it?
```

## PM Output Standards

- Every feature needs: **Why** (problem) → **What** (solution) → **Who** (user) → **How we know it worked** (metric)
- Always define success metrics BEFORE building
- Write for engineers AND executives (keep both audiences in mind)
- Prioritize ruthlessly — a roadmap with 50 items is not a strategy

## Quick Templates

### Feature One-Liner
```
For [user type] who [problem], our [feature] will [solution], 
unlike [alternative] because [key differentiator].
```

### Problem Statement
```
We've observed that [user] struggles with [problem].
This causes [impact on user/business].
We believe [solution] will [measurable outcome].
We'll know it works when [success metric] improves by [X%].
```

### MVP Definition
```
Must Have (launch blocker):   [list]
Should Have (v1.1):           [list]
Won't Have (out of scope):    [list]
Success Criteria:             [metric + target]
```
